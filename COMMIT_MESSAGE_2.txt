feat(backup): дубликаты, ретеншн, rollback, прогресс восстановления, тесты, CI

В дополнение к предыдущему коммиту (профессиональная система бэкапа):

1. Поиск дубликатов (utils/backup_core/diff.py: find_duplicate_entities,
   утечка в публичный API как backup_core.find_duplicates):
   - сканирует роли/категории/каналы сервера на совпадающие имена без
     привязки к какому-либо бэкапу;
   - новые команды: `/find-duplicates`, `L.backup duplicates`.

2. Ретеншн бэкапов (utils/backup_core/storage.py: prune_old_backups,
   MAX_REGULAR_BACKUPS=10, MAX_EMERGENCY_BACKUPS=5):
   - вызывается автоматически после каждого save_backup()/create_emergency_backup();
   - обычные и emergency-бэкапы считаются и обрезаются раздельно.

3. Список бэкапов и выбор конкретного бэкапа:
   - storage.list_backups уже было, добавлено отображение через
     `/backups`, `L.backup list`;
   - `/load` и `/rollback` получили параметр `бэкап` с автодополнением
     (backup_id_autocomplete в cogs/slash.py), без необходимости копировать
     ID руками.

4. Rollback (storage.latest_emergency_backup_id +
   backup_core.latest_emergency_backup_id):
   - `/rollback [бэкап]`, `L.backup rollback [id]` — без аргумента
     откатывается к последнему авто-бэкапу перед прошлым restore;
   - использует тот же RestoreConfirmView/план-превью, что и /load,
     remove_extra=True по умолчанию (смысл rollback — полный возврат).

5. Прогресс восстановления на больших серверах (restore.py:
   apply_plan(..., progress_cb=...), make_throttled_progress_callback):
   - на сервере с сотнями ролей/каналов restore теперь периодически
     обновляет статусное сообщение ("Применяется план: N/M (X%)...");
   - троттлинг по времени (по умолчанию раз в 4 сек), чтобы не упереться
     в rate limit Discord на редактирование сообщений; финальный пункт
     плана всегда показывается, независимо от таймера;
   - подключено и в /load (RestoreConfirmView), и в L.backup restore/rollback.

6. Юнит-тесты (tests/, requirements-dev.txt: pytest):
   - tests/fakes.py — лёгкие фейки Guild/Role/Category/Channel без
     реального discord.py-подключения;
   - tests/test_models.py, test_storage.py, test_diff.py, test_restore.py,
     test_embeds.py — 46 тестов на сравнение/сопоставление по имени,
     конфликты, ретеншн, миграцию легаси-формата, apply_plan, троттлинг
     прогресса, лимит длины embed-полей (тот самый баг с 1024 символами);
   - тесты поймали 2 реальные ошибки до релиза: триггер троттлинга
     прогресса на маленьком time.monotonic() (state["last"] теперь
     инициализируется -inf, а не 0.0) — оставлено в коде как фикс.

7. CI (.github/workflows/ci.yml):
   - py_compile всех .py-файлов + pytest на Python 3.11 и 3.12,
     запускается на каждый push/PR в main.

8. Документация (README.md, cogs/help.py):
   - таблица слэш-команд для бэкапа;
   - обновлены описания L.backup и /load/L.backup restore под новые
     сабкоманды и поведение;
   - раздел "Разработка и тесты" с инструкцией по pytest.
